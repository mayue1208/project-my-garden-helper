# 植物养护助手 · 微信小程序设计文档

## 1. 概述

一款面向家庭场景的植物养护记录微信小程序，支持一人加入多个家庭，家庭成员协作记录养护操作、设置提醒、追踪植物成长历程。

## 2. 技术选型

| 维度 | 选择 |
|------|------|
| 前端框架 | uni-app（Vue 语法） |
| 后端 | 微信云开发（云函数 + 云数据库 + 云存储） |
| 状态管理 | Vuex/Pinia |
| 用户体系 | 微信授权登录（openId + 微信昵称/头像） |

## 3. 用户登录流程

利用微信小程序的开屏授权能力，实现一键登录：

```
用户首次打开小程序
        ↓
展示「微信一键登录」开屏页
（显示微信头像+昵称预览）
        ↓
用户点击「允许」
    → wx.login() 获取 code
    → 云函数接收 code，通过 openId 创建/查找用户
    → 获取微信昵称和头像（chooseAvatar）
    → 写入 users 集合（openId, nickName, avatarUrl）
        ↓
进入首页（已登录状态）
        ↓
后续启动 → 检查 openId 是否存在 → 自动登录
```

- 无需注册账号密码
- 微信头像和昵称可直接复用（用户也可在「我的」页面修改）
- 通过 `open-data` 组件直接展示微信昵称和头像，无需额外存储

## 5. 云数据库设计

### 核心集合

```
plants                     // 植物表
  _id: string
  name: string             // 植物名称
  nickname: string         // 昵称
  species: string          // 品种
  roomId: string           // 所属房间 ID
  familyId: string         // 所属家庭 ID
  photo: string            // 云存储文件 ID
  purchaseDate: Date       // 购入日期
  status: string           // 'healthy'|'warning'|'critical'|'dead'|'given'
  note: string             // 备注
  createdBy: string        // 创建者 openId
  createdAt: Date
  updatedAt: Date

rooms                      // 房间表
  _id: string
  name: string
  familyId: string
  sortOrder: number
  isDefault: boolean       // 是否为预置房间
  createdAt: Date

care_configs               // 养护周期配置表（1植物N条）
  _id: string
  plantId: string
  familyId: string
  type: string             // 'water'|'fertilize'|'prune'|'pesticide'|'clean'
  intervalDays: number     // 间隔天数
  lastTime: Date           // 上次养护时间
  nextTime: Date           // 下次提醒时间
  enabled: boolean         // 是否开启提醒
  createdBy: string

care_records               // 养护记录表
  _id: string
  plantId: string
  familyId: string
  type: string             // 'water'|'fertilize'|'prune'|'repot'|'pesticide'|'clean'
  recordedBy: string       // 操作人 openId
  note: string
  createdAt: Date

growth_events              // 成长事件表
  _id: string
  plantId: string
  familyId: string
  type: string             // 'repot'|'prune'|'pest'|'bloom'|'propagate'|'other'
  description: string
  photos: string[]         // 云存储文件 ID 数组
  eventDate: Date
  createdBy: string
  createdAt: Date

memorials                  // 纪念碑表
  _id: string
  plantId: string          // 引用原植物 ID
  plantName: string        // 冗余：植物名
  plantPhoto: string       // 冗余：植物照片
  type: string             // 'dead'|'given'
  deathDate: Date          // 死亡/赠出日期
  reason: string           // 原因
  farewell: string         // 告别语
  recipient: string        // 赠予对象（type='given'时）
  memorialPhotos: string[] // 纪念照片
  createdBy: string
  familyId: string
  createdAt: Date

families                   // 家庭表
  _id: string
  name: string
  inviteCode: string       // 邀请码（6位）
  createdBy: string
  createdAt: Date

family_members             // 家庭成员表
  _id: string
  familyId: string
  userId: string           // 用户 openId
  role: string             // 'admin'|'member'
  joinedAt: Date

delayed_reminders          // 延迟提醒表（用户延后的提醒）
  _id: string
  plantId: string
  familyId: string
  type: string             // 'water'|'fertilize'|'prune'|'pesticide'|'clean'
  remindAt: Date           // 延后到的具体时间
  createdBy: string
  createdAt: Date
```

### 云数据库安全规则

- plants/rooms/care_records/growth_events/memorials：仅 familyId 匹配的成员可读写
- families：仅管理员可修改名称，所有成员可读
- family_members：仅管理员可管理成员

## 6. 页面详细设计

### 5.1 首页 - 我的植物

**全局家庭切换器**（所有数据页顶部共享）：
- 首页/提醒/纪念 页面顶部固定栏显示当前家庭名称 + 下拉箭头
- 点击展开用户所属的所有家庭列表，选中即切换当前数据视角
- 切换后，所有页面的数据（植物/提醒/纪念）自动刷新为所选家庭的数据

```
┌─────────────────────────┐
│ [我家 ↕]   房间：[全部 ▼] │  ← 左：家庭切换，右：房间筛选
├─────────────────────────┤
│  ...植物卡片...           │
└─────────────────────────┘
```

**功能**：
- 顶部下拉筛选房间（默认「全部」）
- 按房间分组展示植物卡片
- 每张卡片展示：植物名、状态标识（健康/黄叶/病害）、距上次浇水天数
- 底部概览栏：植物总数、今日需浇水数量
- 悬浮 [+] 按钮：添加新植物

**状态标签**：
| 标签 | 含义 | 条件 |
|------|------|------|
| ● 健康 | 正常生长 | 无异常记录 |
| ⚠️ 黄叶/长虫 | 需关注 | 有生长事件含相关类型 |
| 🔴 病危 | 严重问题 | 管理员手动标记 |

### 5.2 添加/编辑植物

**字段**：植物名称、品种（常用植物库可选）、昵称、所属房间（下拉选择）、购入日期、养护周期（选填，后续可改）、备注

**交互**：拍照或从相册选择，支持裁剪

### 5.3 植物详情

**信息区**：照片、品种、昵称、房间、购入日期、状态
**养护周期区**：各类型（浇水/施肥/修剪/杀虫）的频率开关与间隔天数设置
**最近养护区**：最近3条养护记录（谁在什么时间做了什么）
**成长记录区**：最近4条成长事件，点击「查看完整时间线」进入时间线页

### 5.4 成长记录时间线

**布局**：纵向时间线，按年月分组
**内容**：每个节点显示照片、事件类型、描述文字、日期
**交互**：点击右上角 [+] 添加新事件，选择类型（换盆/修剪/长虫/开花/繁殖/其他）、拍照、写描述

### 5.5 养护提醒

**筛选维度**：
- 空间：顶部下拉选择房间（默认当前选中或「全部」）
- 时间：Tab 切换「今日」「7天」

**列表分组**：
- 📅 今日（含逾期）：逾期项以 ⚠️ 标识并显示逾期天数
- 📅 明天
- 📅 后续日期（7天内）

**每项展示**：

```
🪴 龟背竹
💧 浇水（上次：3天前）
[已浇]  [延后 ⏰]
```

**交互按钮**（两条操作路径）：

**① [已浇] — 一键完成**
→ 写入 care_records → 更新 care_config.lastTime/nextTime → 从列表消失

**② [延后 ⏰] — 延迟提醒**
→ 弹出选择器，选项：
- 1 小时后
- 今天晚些时候（18:00）
- 明天
- 后天
- 一周后
- 一个月后
- 两个月后
- 自定义（选择日期时间）
→ 写入 delayed_reminders（plantId, type, remindAt）→ 该提醒从列表暂时消失
→ remindAt 到达时，该提醒重新出现在对应日期分组中（带「已延后」标记）

**规则**：
- 同一 plantId + type 只能有一条活跃的延迟提醒，再次延后覆盖原记录
- 延后到达的提醒同样支持 [已浇] 和再次 [延后]
- 手动完成养护后（[已浇]），对应的延迟提醒记录自动删除

**快速记录浮层**：点击提醒页顶部 `[📝+]` 按钮弹出，选择植物、操作类型、备注，提交

### 5.6 纪念碑

**分区**：已故 / 已赠出
**卡片**：照片、植物名、生命周期（购入日-死亡日）、死因、告别语
**统计**：存活率（存活数/总数）
**移入流程**：选择植物 → 选择原因（死亡/赠出）→ 填写日期/原因/告别语/纪念照 → 确认 → 从 plants 移至 memorials

### 5.7 我的

**信息**：头像、昵称
**我的家庭列表**：

```
我的家庭
├─ 🏠 我家（3位家人）  ← 当前选中
├─ 🏠 爸妈家（2位家人）
└─ [+ 加入新家庭]
```

- 每个家庭卡片显示：家庭名、成员数、当前角色（管理员/成员）
- 点击进入该家庭的「家庭管理」页
- 支持「加入新家庭」（输入邀请码）
- 底部「创建新家庭」

**功能入口**：
- 植物统计（养护总数、存活率、最勤快家人等）
- 设置

### 5.8 家庭管理

从「我的」页面点击某个家庭卡片进入该家庭的独立管理页：

```
┌─────────────────────────────┐
│  ← 家庭管理 · 我家            │
├─────────────────────────────┤
│  家庭名：[我家的植物园] [修改]   │
│  邀请码：[AB12CD]  [分享]     │
├─────────────────────────────┤
│  家人 (3)                     │
│  ├─ 🧑 我      [管理员]       │
│  ├─ 👩 妈妈    [成员]         │
│  └─ 👨 爸爸    [成员]         │
├─────────────────────────────┤
│  最近操作                      │
│  ├─ 妈妈 · 10:30   [查看→]    │
│  ├─ 爸爸 · 昨天    [查看→]    │
│  └─ 我   · 昨天    [查看→]    │
└─────────────────────────────┘
```

**功能**：
- 编辑家庭名（仅管理员）
- 查看/分享邀请码（仅管理员）
- 成员列表（显示角色）
- 退出该家庭（仅普通成员，管理员需先转让身份）
- 最近操作列：谁几点做了什么（不展示具体植物）
- 每条操作右侧 [查看] → 进入详情页，按操作类型分组展示操作了哪些植物

**加入家庭流程**：在「我的」页面点击「加入新家庭」→ 输入6位邀请码 → 自动成为普通成员

### 5.9 设置

- 提醒推送时间（默认 09:00）
- 通知开关（浇水/施肥/逾期提醒独立开关）
- 关于 / 版本号

### 5.10 房间管理

- 预置房间：客厅、阳台、卧室、书房、厨房、卫生间
- 支持自定义新增/重命名/删除
- 首页按房间筛选时支持「全部」选项
- 提醒页按房间筛选时支持「全部房间」选项
- 在植物详情页可修改植物所属房间

## 7. 数据流转

### 养护提醒计算

```
用户设置 care_config（植物A，浇水，每7天）
        ↓
查询时实时计算
        ↓
Step 1: 查询所有 enabled=true 的 care_config
    → nextTime <= now + 7days
    → 按 nextTime 排序分组
        ↓
Step 2: 查询 delayed_reminders 中 remindAt <= now + 7days 的记录
    → 与 Step 1 结果合并去重（同一 plantId+type 取延迟时间）
        ↓
返回给前端展示
        ↓
用户点击 [已浇]
    → 写入 care_records（plantId，type，recordedBy=userId）
    → 更新 care_config.lastTime = now
    → 更新 care_config.nextTime = now + intervalDays
    → 删除对应 delayed_reminders 记录（如有）
        ↓
用户点击 [延后]
    → 弹出延时选项选择器
    → 写入/覆盖 delayed_reminders（plantId, type, remindAt）
    → 该条提醒暂时从列表消失
        ↓
remindAt 到达时
    → 该提醒重新出现在对应日期分组
    → 标记为「⏰ 已延后」
    → 交互按钮不变：仍可 [已浇] 或再次 [延后]
```

### 家庭数据隔离（一人多家）

```
用户登录 → 获取 openId
    → 查询 family_members 找到用户所属的所有家庭
    → 返回家庭列表供用户选择
        ↓
用户在当前会话中选定一个「当前家庭」
    → 所有数据查询均带当前 familyId 条件
    → 切换家庭时，全局状态更新 familyId，所有页面数据刷新
        ↓
用户可随时在页面顶部切换家庭
    → 跨家庭的植物/提醒/纪念完全隔离
    → 养护操作、添加植物等均写入当前家庭
```

### 移入纪念碑

```
用户操作「移入纪念碑」
    → 写入 memorials（含原植物信息快照）
    → 更新 plants.status = 'dead'|'given'
    → 植物从首页列表隐藏，出现在纪念碑
```

## 8. 边界情况与错误处理

### 数据边界
- 植物名称为必填，最长 20 字符
- 昵称最长 20 字符
- 照片最多 9 张/条成长事件
- 纪念告别语最多 200 字符
- 房间名最长 10 字符，最多 10 个自定义房间

### 延迟提醒边界
- 同一植物+同类型只能有一条活跃延迟提醒，再次延后自动覆盖
- 延后时间不得早于当前时间
- 若用户在延后期间手动完成了养护（[已浇]），自动清除对应延迟提醒
- 延后到达的提醒若同时到了 regular 的提醒日，以免打扰原则：仅显示一条，优先使用延后时间
- 延迟时间不限，可选 1小时/今晚/明天/后天/一周/一个月/两个月/自定义日期

### 空状态
- 无植物时：首页显示「还没有植物，点击 + 添加第一棵吧」
- 无提醒时：「所有植物都照顾好了，真棒！」
- 无成长事件时：「还没有记录，开始记录植物的成长吧」
- 纪念碑为空时：「还没有植物离开」
- 家人为空时：「邀请家人一起照顾植物吧」
- 成员操作无记录时：「暂无操作记录」

### 错误处理
- 云函数调用失败：toast「网络开小差了，请稍后重试」
- 图片上传失败：toast「图片上传失败」+ 本地暂存重试
- 邀请码无效：toast「邀请码无效，请确认后重试」
- 权限不足：toast「只有管理员才能执行此操作」
