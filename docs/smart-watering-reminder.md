# 植物浇水智能提醒系统

## 功能说明

本系统实现了智能浇水提醒功能，能够根据用户的实际浇水行为动态调整提醒间隔，同时每3个月重置回初始设置。

### 核心特性

1. **智能间隔调整**：
   - 如果用户提前浇水，系统会根据实际间隔调整下次提醒时间
   - 如果用户延迟浇水，系统会适当延长下次提醒时间

2. **定期重置**：
   - 每3个月（约90天）系统会自动重置提醒间隔到初始设置值
   - 避免长期积累导致的过度偏离

3. **特定适用范围**：
   - 此智能逻辑仅适用于"浇水"提醒
   - 施肥、修剪等其他操作仍使用固定间隔

### 算法逻辑

#### 智能调整规则：
- **提前浇水**：下次提醒间隔 = 实际间隔天数（从上次浇水到本次浇水的天数）
- **延迟浇水**：下次提醒间隔 = 原间隔 + 延迟天数（超出预期时间的天数）
- **边界限制**：提醒间隔限制在3-30天之间

#### 重置逻辑：
- 每90天检查一次，若超过90天未重置，则重置为初始设置的天数

## 数据库结构变更

在`care_configs`集合中新增以下字段（仅针对浇水配置）：

- `initialIntervalDays`: number - 初始设置的间隔天数
- `actualIntervalDays`: number - 当前实际使用的间隔天数
- `lastResetDate`: Date - 最后一次重置的日期

## 部署说明

### 1. 部署云函数

将以下云函数部署到微信云开发：
- `care` - 处理智能浇水提醒逻辑
- `plant` - 创建植物时自动设置初始字段
- `data_migrate` - 数据迁移（仅首次升级时使用）

### 2. 数据迁移（首次升级必做）

如果系统已有数据，需要运行数据迁移脚本来为现有浇水配置添加新字段：

1. 部署`data_migrate`云函数
2. 调用云函数执行迁移：
   ```javascript
   wx.cloud.callFunction({
     name: 'data_migrate',
     data: {
       action: 'migrate-water-configs'
     }
   })
   ```

### 3. 注意事项

- 系统会自动为新添加的植物设置初始字段
- 只有`type: 'water'`的配置会受到智能调整逻辑的影响
- 其他养护类型（施肥、修剪等）继续使用固定间隔逻辑

## 使用效果示例

假设初始设置为每5天浇水：

**场景1 - 提前浇水**：
- 第1次：第5天时发现植物需要浇水，提前1天浇水
- 第2次：下次提醒将在4天后（根据实际间隔调整）

**场景2 - 延迟浇水**：
- 第1次：第5天没浇水，第7天才浇水（延迟2天）
- 第2次：下次提醒将在7天后（原间隔5天 + 延迟2天）

**场景3 - 3个月重置**：
- 即使经过多次调整，3个月后会自动重置为初始设置的天数
